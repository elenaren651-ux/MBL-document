这个仓库当前以“原样整理上传”为目标，保留现有 1:1 原型、Axure 导出资源和 PRD 文档，方便：

- 在 GitHub 上存档和协作
- 继续迭代当前原型
- 后续逐步重构为正式前端项目

## 本地预览

这是一个静态原型项目，不需要安装依赖。

直接打开下面的文件即可：

- `MBL制单原型_1比1入口.html`

如果需要本地服务预览，可以在仓库目录运行任意静态文件服务，例如：

```bash
python3 -m http.server 4173
```

然后访问：

- `http://localhost:4173/MBL%E5%88%B6%E5%8D%95%E5%8E%9F%E5%9E%8B_1%E6%AF%941%E5%85%A5%E5%8F%A3.html`

## 后续重构建议

建议采用“保留原型基线 + 新建正式前端目录”的方式推进，例如：

1. 保留当前文件结构不动，作为验收和对照基线
2. 新建 `app/` 或 `src/` 目录，开始 React/Vite 重构
3. 优先重构入口导航、列表页、详情页等核心页面
4. 每次重构都对照 `mbl-prototype-1to1/` 做视觉和交互校验

这样做的好处是：

- 不会破坏当前可用原型
- 可以分阶段迁移
- Git 历史会同时保留“原型版本”和“正式代码版本”
