## 需求背景

当前提单在执行「提交船公司」后，系统会根据船司返回结果生成草稿件和 VC 件。

当同一个提单号存在多次提交船公司的情况时，操作人员需要追溯每一次提交动作的提交结果、失败原因，以及当次提交时页面填写内容的快照信息。当前页面缺少面向结果物的提交历史入口，导致操作人员排查失败原因或确认当前草稿件/VC 件来源时，需要依赖人工沟通或跨页面查询。

本次需求在草稿件和 VC 件页面增加「提交记录」入口，支持用户查看同一提单号下的多次提交船公司记录，并通过「查看截图」回溯当次提交时的页面状态。

## 适配范围

- 适配船司：全船司
- 适配客户：全客户
- 外部系统：无

## 产品需求

| # | 作为 | 我想要 | 以便于 |
|---|---|---|---|
| 1 | 操作人员 | 在草稿件和 VC 件中查看提交记录 | 快速确认当前草稿件/VC 件来源于哪一次提交船公司动作 |
| 2 | 操作人员 | 查看每次提交的成功/失败状态和失败原因 | 排查提交失败原因，减少重复沟通 |
| 3 | 操作人员 | 点击查看当次提交截图 | 回溯提交时页面字段、箱号、提单号等关键信息 |
| 4 | 操作人员 | 仅在存在多次提交时看到提交记录入口 | 避免单次提交场景产生不必要的信息干扰 |

## 功能及界面设计

### 1. 草稿件和 VC 件增加「提交记录」入口

- 入口页面：
  - 制单详情页 - 草稿件 tab。
  - 制单详情页 - VC 件 tab。
- 入口位置：
  - 草稿件 tab 和 VC 件 tab 内容区右上角，全局固定展示在当前 tab 结果物区域内。
  - 与「提交人、提交时间、回执时间、文件、比对状态」等结果物信息同层级展示。
  - 入口不放在页面顶部工具栏，不作为独立原型入口展示。
- 入口文案：
  - 默认文案为「提交记录」。
  - 当提交次数大于 1 时，展示次数标识，例如「提交记录 2」「提交记录 3」。
- 展示逻辑：
  - 当当前提单号执行「提交船公司」次数 > 1 时，展示「提交记录」入口。
  - 当当前提单号执行「提交船公司」次数 <= 1 时，不展示「提交记录」入口。
  - 提交次数仅统计当前提单号维度下的「提交船公司」动作，不按草稿件/VC 件分别统计。
- 权限规则：
  - 可访问草稿件或 VC 件页面的操作人员，可查看该提单号的提交记录。
  - 本期不新增独立权限点。

### 2. 点击「提交记录」展示提交记录弹框

- 触发方式：
  - 点击「提交记录」入口后，打开「提交记录」弹框。
- 弹框标题：
  - 「提交记录」。
- 排序规则：
  - 按提交次数倒序展示。
  - 最新一次提交展示在最上方。
- 每条记录展示字段：

| 字段 | 是否展示 | 说明 |
|---|---|---|
| 提交次数 | 是 | 展示为「第1次」「第2次」「第3次」等 |
| 提交时间 | 是 | 执行提交船公司的时间 |
| 操作人 | 是 | 执行提交船公司的用户 |
| 提交状态 | 是 | 成功 / 失败 |
| 失败原因 | 条件展示 | 仅提交状态为失败时展示 |
| 查看截图 | 是 | 点击后查看当次提交时页面快照 |

- 交互规则：
  - 点击右上角关闭按钮，关闭弹框。
  - 点击「取消」，关闭弹框。
  - 点击「确定」，关闭弹框。
  - 点击某条记录的「查看截图」，查看当次提交截图。
- 弹框视觉规则：
  - 弹框居中展示，背景页面增加半透明黑色遮罩。
  - 记录使用时间线样式展示，左侧展示「第 N 次」标签。
  - 最新一次提交使用紫色标签高亮。
  - 成功状态使用绿色圆点，失败状态使用红色圆点。
  - 失败原因使用红色文字展示在失败记录下方。
- 空状态：
  - 由于入口仅在提交次数 > 1 时展示，正常情况下弹框内至少存在 2 条记录。
  - 若因数据异常导致记录为空，弹框展示「暂无提交记录」。

### 3. 点击「查看截图」展示提交截图

- 触发方式：
  - 在提交记录弹框中点击某条记录右侧的「查看截图」。
- 展示内容：
  - 展示当次点击「提交船公司」时的页面状态。
  - 页面快照为只读。
  - 不支持编辑。
- 快照用途：
  - 用于追溯当次提交时的字段内容、箱号、提单号、船司账号、收发货人等关键信息。
- 交互规则：
  - 截图预览支持关闭。
  - 关闭截图预览后，返回提交记录弹框。
  - 截图预览不影响提交记录弹框内的排序和内容。
- 异常状态：
  - 若页面快照加载失败，展示「截图加载失败，请稍后重试」。
  - 若该记录缺少页面快照，展示「暂无截图」。

### 4. 提交记录生成逻辑

- 记录生成时机：
  - 用户点击「提交船公司」，并完成一次提交请求后，系统生成一条提交记录。
  - 无论提交结果为成功或失败，均生成提交记录。
- 记录范围：
  - 提单号级别。
  - 同一个提单号下，多次「提交船公司」动作归集到该提单号的提交记录中。
- 提交次数规则：
  - 按同一提单号下已生成的提交记录顺序递增。
  - 第一次提交为「第1次」，第二次提交为「第2次」，以此类推。
- 记录内容：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 提单号 | 是 | 当前提交对应的提单号 |
| 提交次数 | 是 | 当前为第几次提交 |
| 提交时间 | 是 | 提交动作发生时间 |
| 操作人 | 是 | 当前操作账号 |
| 提交状态 | 是 | 成功 / 失败 |
| 失败原因 | 条件必填 | 提交失败时必填；提交成功时不展示 |
| 页面快照 | 是 | 记录当次提交时页面状态 |

## 数据与状态规则

- 数据维度：按提单号维度记录和查询。
- 数据范围：查询当前提单号下全部提交船公司记录。
- 排序规则：提交次数倒序，最新提交记录在上。
- 入口可见性：仅当当前提单号提交记录数量 > 1 时展示。
- 失败原因：仅失败记录展示；成功记录不展示失败原因字段。
- 页面快照：以提交动作发生时的页面状态为准，后续页面字段变化不反向影响历史快照。
- 实时性：用户完成一次提交请求后，提交记录应可在当前详情页内刷新或重新打开弹框时查看。

## 验收标准

1. Given 当前提单号仅提交船公司 1 次，When 用户进入草稿件 tab 或 VC 件 tab，Then 页面不展示「提交记录」入口。
2. Given 当前提单号提交船公司次数大于 1，When 用户进入草稿件 tab 或 VC 件 tab，Then 页面右上角展示「提交记录 N」入口，N 为当前提单号提交次数。
3. Given 用户点击「提交记录」入口，When 弹框打开，Then 弹框标题为「提交记录」，并按提交次数倒序展示记录。
4. Given 某条提交记录状态为失败，When 用户查看该条记录，Then 该条记录展示失败原因。
5. Given 某条提交记录状态为成功，When 用户查看该条记录，Then 该条记录不展示失败原因字段。
6. Given 用户在提交记录弹框点击「查看截图」，When 截图预览打开，Then 系统展示该次提交时的只读页面快照。
7. Given 用户关闭截图预览，When 截图预览关闭，Then 用户仍停留在提交记录弹框中。
8. Given 用户点击弹框右上角关闭按钮、取消或确定，When 操作完成，Then 提交记录弹框关闭。
9. Given 用户执行一次提交船公司动作且请求完成，When 系统记录提交结果，Then 当前提单号新增一条提交记录，并包含提单号、提交次数、提交时间、操作人、提交状态和页面快照。
10. Given 提交船公司失败，When 系统生成提交记录，Then 失败原因字段必填并在提交记录弹框中展示。

2. 「提交次数」是否需要跨历史迁移数据重新计算，还是仅对新生成记录生效？
3. 操作人展示使用账号名、用户姓名，还是账号名 + 用户姓名？
4. 截图预览是否需要支持下载、放大、缩小或打开新窗口？
5. 提交记录入口展示在草稿件和 VC 件时，是否都展示同一提单号的全部提交记录，还是只展示与当前结果物有关的记录？
