# 【MSK】MBL制单-邮件回执接收方

## 1. Summary

本需求用于支持 MSK 官网的“邮件回执接收方”能力。操作员在 MBL 制单详情页填写该字段后，提交船公司时平台将该字段传给通道侧，由通道在 MSK 官网填入对应位置，以便指定邮箱收到后续草稿件或 VC 件相关回执。

## 2. Contacts

| 角色 | 关注点 | 备注 |
|---|---|---|
| 业务方 | 指定邮箱能收到 MSK 官网回执 | 需要与官网字段规则一致 |
| 操作员 | 在制单页面可填写和修改接收方 | 需要清楚知道填写格式和数量限制 |
| 平台测试 | 校验字段展示、保存、提交和传参 | 需要覆盖正常和异常输入 |
| 通道侧 | 接收平台字段并提交 MSK 官网 | 需确认字段名、分隔符和最大数量 |
| 产品/研发 | 实现字段、校验、保存、提交映射和日志 | 需保持前后端规则一致 |

## 3. Background

MSK 官网在提交提单时提供“邮件回执接收方”字段。该字段填写后，官网会把后续对应的 VC 件或回执邮件发送给填写的邮箱。官网最多支持填写 5 个邮箱，多个邮箱之间使用中文分号“；”隔开。

当前平台制单页面没有体现该字段。操作员无法在平台内指定回执接收方，提交船公司时也无法把该信息传给通道侧。业务方希望在制单页面增加该字段，并在提交船公司时传给通道侧做测试和联调。

## 4. Objective

目标是让 MSK 单据可以在平台内维护邮件回执接收方，并在提交船公司时准确传给通道侧。

关键结果：

- MSK 制单详情页支持填写“邮件回执接收方”。
- 字段支持 1 到 5 个邮箱，多个邮箱用中文分号“；”隔开。
- 保存和提交船司前可以拦截邮箱格式错误、数量超过 5 个、分隔符错误等问题。
- 提交船公司时，平台把该字段传给通道侧，通道侧可以按 MSK 官网规则填入。
- 操作员可以在字段变更后看到保存结果，后续重新进入详情页仍可看到已保存内容。

## 5. Market Segment(s)

本需求面向处理 MSK MBL 制单的操作员、平台测试和通道侧对接人员。

适用范围：

- 船司：MSK。
- 页面：MBL 制单详情页、待提交船司、提交船司前校验、提交船公司中、已提交船公司后的只读详情。
- 数据对象：邮件回执接收方。
- 外部系统：MSK 官网、船司通道。

暂不纳入范围：

- 非 MSK 船司是否支持邮件回执接收方。
- VC 件发送客户状态的逻辑改造。
- 自动从客户资料、联系人、历史邮件中推荐邮箱。
- 通道侧对 MSK 官网之外其他系统的适配。

## 6. Value Proposition(s)

对操作员：

- 可以在平台内一次性维护回执接收方，不需要额外联系通道侧人工处理。
- 字段错误会在保存或提交前被提示，减少提交船司失败。
- 已提交后可以查看当时传给通道侧的接收方，方便排查。

对业务方：

- MSK 回执邮件可以发送给指定人员。
- 减少 VC 件或回执邮件只到默认邮箱导致的转发和沟通成本。
- 平台字段与官网规则一致，降低漏填和误填风险。

对平台和通道：

- 平台保存结构化字段，通道按契约提交官网。
- 提交报文可追踪，便于通道测试和问题定位。
- 前后端统一校验规则，避免无效邮箱进入提交链路。

## 7. Solution

### 7.1 UX/Prototypes

在 MSK 制单详情页增加字段：

| 字段 | 展示位置 | 控件 | 是否必填 | 说明 |
|---|---|---|---|---|
| 邮件回执接收方 | 建议放在右侧制单信息表单中，与提交船司相关信息同组 | 多行文本框或可横向展示的输入框 | 否 | 最多 5 个邮箱，多个邮箱用中文分号“；”隔开 |

交互规则：

1. 字段仅在 MSK 单据展示；非 MSK 单据不展示。
2. 字段默认非必填。为空时允许保存和提交船公司。
3. 输入框占位文案建议为：“请输入邮箱，多个邮箱用；隔开，最多 5 个”。
4. 字段失焦、保存制单信息、提交船公司前均触发校验。
5. 校验失败时，字段展示错误态，并在字段下方展示明确错误文案。
6. 进入提交船公司中、已提交船公司、草稿件已出、VC 已出、已关闭等不可编辑状态时，字段只读展示。
7. 提交船公司快照和提交记录中应展示本次提交的“邮件回执接收方”，方便后续排查。

错误文案建议：

| 场景 | 提示文案 |
|---|---|
| 邮箱格式错误 | 邮件回执接收方邮箱格式错误，请检查后重新填写 |
| 超过 5 个邮箱 | 邮件回执接收方最多填写 5 个邮箱 |
| 使用英文分号或逗号分隔 | 多个邮箱请使用中文分号“；”隔开 |
| 出现连续分号或空邮箱 | 请删除多余的分号或空邮箱 |

### 7.2 Key Features

#### 7.2.1 字段新增

系统为 MSK 单据新增“邮件回执接收方”字段。

建议内部字段：

| 字段 | 类型 | 示例 | 说明 |
|---|---|---|---|
| receiptEmailRecipients | string | a@test.com；b@test.com | 页面展示和保存的原始字符串 |
| receiptEmailRecipientList | string[] | ["a@test.com", "b@test.com"] | 后端或通道提交时可使用的解析结果 |

保存原则：

- 前端展示时保留中文分号“；”。
- 保存前去除每个邮箱前后的空格。
- 不保留首尾分号。
- 不允许连续分号产生空邮箱。
- 如果用户粘贴内容含换行，应提示错误或自动替换为空格后再校验，具体由研发按现有输入框规范决定。

#### 7.2.2 校验规则

字段为空时不校验邮箱格式，允许保存和提交。

字段非空时，按以下规则校验：

| 校验项 | 规则 |
|---|---|
| 分隔符 | 多个邮箱必须使用中文分号“；”分隔 |
| 数量 | 最少 1 个，最多 5 个 |
| 空值 | 不允许首尾分号、连续分号或分隔后为空的邮箱 |
| 邮箱格式 | 每个邮箱必须符合常规邮箱格式 |
| 空格 | 每个邮箱前后的空格自动 trim，中间空格不允许 |

建议前端正则：

```regex
^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}(；[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}){0,4}$
```

说明：

- 该正则用于前端快速提示，限制最多 5 个邮箱。
- 该正则只接受中文分号“；”。
- 后端不能只依赖前端正则。后端提交前需要按“；”拆分后逐个校验邮箱格式和数量。
- 若平台已有统一邮箱校验工具，应优先复用统一工具，并在外层增加 MSK 的数量和分隔符规则。

#### 7.2.3 保存规则

保存制单信息时：

1. 字段为空：保存为空。
2. 字段非空且校验通过：保存规范化后的字符串。
3. 字段非空但校验失败：不允许保存，并提示具体错误。
4. 修改字段后需要记录字段变更日志。

日志内容建议：

- 操作类型：修改邮件回执接收方。
- 操作内容：原值、新值。
- 操作人、操作时间。

#### 7.2.4 提交船司前校验

点击“提交船公司”或“强制提交船司”时，系统需要再次校验“邮件回执接收方”。

处理规则：

| 场景 | 是否允许提交 | 说明 |
|---|---|---|
| 字段为空 | 允许 | MSK 官网字段非必填 |
| 1 到 5 个合法邮箱 | 允许 | 提交给通道侧 |
| 超过 5 个邮箱 | 不允许 | 提示最多 5 个 |
| 邮箱格式错误 | 不允许 | 提示邮箱格式错误 |
| 分隔符不是中文分号 | 不允许 | 提示使用中文分号 |

#### 7.2.5 提交通道映射规则

提交船公司时，平台将字段传给通道侧。

建议映射：

| 平台字段 | 平台值 | 通道字段 | 通道值 |
|---|---|---|---|
| 邮件回执接收方 | 空 | receiptEmailRecipients / 待通道确认 | 空字符串或 null |
| 邮件回执接收方 | a@test.com | receiptEmailRecipients / 待通道确认 | a@test.com |
| 邮件回执接收方 | a@test.com；b@test.com | receiptEmailRecipients / 待通道确认 | a@test.com；b@test.com |

推荐原则：

- 平台传给通道侧的分隔符保持中文分号“；”。
- 不把邮箱数组拼成英文分号或逗号。
- 不自动补充默认邮箱，除非后续有明确客户规则。
- 通道侧字段名、空值格式以通道契约为准。

#### 7.2.6 提交记录与只读展示

提交船公司成功或进入提交船公司中后：

- 页面只读展示本次提交的邮件回执接收方。
- 提交记录中展示该字段的提交值。
- 若通道返回提交截图，截图仍按现有方式展示；字段值以平台提交记录为准。

### 7.3 Technology

前端：

- MSK 制单详情页新增字段。
- 失焦、保存、提交前执行校验。
- 错误态和提示文案按现有表单规范展示。
- 不可编辑状态下只读展示字段。

后端：

- 数据模型新增邮件回执接收方字段。
- 保存接口支持字段保存和日志记录。
- 保存和提交前复用同一套校验规则。
- 提交船公司接口将字段映射给通道侧。
- 提交记录保存本次提交的字段快照。

通道：

- 接收平台传入的邮件回执接收方。
- 在 MSK 官网对应字段填入该值。
- 支持空值、单邮箱和最多 5 个邮箱。
- 若官网或通道侧对字段名、分隔符、空值有特殊要求，需要在联调中回写到本 PRD。

### 7.4 Assumptions

- MSK 官网“邮件回执接收方”字段为非必填。
- MSK 官网最多支持 5 个邮箱。
- 多个邮箱之间使用中文分号“；”隔开。
- 通道侧可以接收平台传入的原始字符串，并按官网规则填入。
- 字段只影响 MSK 官网回执邮件接收，不改变平台内部 VC 件监听、VC 件下载和 VC 发送客户逻辑。

## 8. Release

### V1 范围

- MSK 制单详情页新增“邮件回执接收方”字段。
- 字段支持为空、单邮箱、最多 5 个邮箱。
- 多邮箱使用中文分号“；”分隔。
- 保存和提交船司前做前后端校验。
- 提交船公司时传给通道侧。
- 提交记录和只读详情展示该字段。
- 字段变更记录操作日志。

### V2 可选优化

- 从客户资料或历史提交记录中带出默认回执接收方。
- 支持邮箱标签化输入，减少分号输入错误。
- 支持从往来邮件发件人、抄送人中快捷选择邮箱。
- 增加客户级配置，指定某客户 MSK 单据默认回执接收方。

## 验收标准

1. MSK 单据制单详情页展示“邮件回执接收方”字段。
2. 非 MSK 单据不展示该字段。
3. 字段为空时，可以保存并提交船公司。
4. 输入 1 个合法邮箱时，可以保存并提交船公司。
5. 输入 2 到 5 个合法邮箱，且使用中文分号“；”分隔时，可以保存并提交船公司。
6. 输入超过 5 个邮箱时，保存和提交船公司均失败，并提示“邮件回执接收方最多填写 5 个邮箱”。
7. 输入邮箱格式错误时，保存和提交船公司均失败，并提示邮箱格式错误。
8. 使用英文分号、英文逗号、中文逗号等分隔多个邮箱时，保存和提交船公司均失败，并提示使用中文分号“；”隔开。
9. 输入首尾分号、连续分号或空邮箱时，保存和提交船公司均失败。
10. 提交船公司时，平台传给通道侧的字段值与页面规范化后的展示值一致。
11. 提交记录中可以查看本次提交的“邮件回执接收方”。
12. 已提交船公司、提交船公司中、草稿件已出、VC 已出、已关闭等不可编辑状态下，该字段只读展示。
13. 修改字段并保存后，系统记录操作日志，包含原值、新值、操作人和操作时间。

## 待确认问题

1. 通道侧最终字段名是什么。
2. 字段为空时，通道侧希望接收空字符串、null，还是不传该字段。
3. MSK 官网是否严格只接受中文分号“；”，还是英文分号也可接受；V1 建议按业务描述只支持中文分号。
4. 官网回执范围是否仅包含 VC 件，还是包含草稿件、提交结果、异常回执等全部邮件。
5. 是否需要在“提交船公司”弹框或快照中额外高亮展示该字段。
