# mbl-result-object-modeler

## 1. 目标

统一定义预览件、草稿件、VC件、提交记录等结果物的业务模型和前端呈现结构。

## 2. 适用场景

- 设计结果物区域
- 设计提交记录弹框
- 设计回执与文件展示
- 设计结果物归属规则

## 3. 输入

- 结果物类型
- 来源渠道
- 展示需求
- 适用船司规则

## 4. 输出

- 结果物模型
- 展示结构建议
- 归属与命名规则
- 页面入口建议

## 5. 工作流程

1. 识别结果物类型
2. 明确生成来源与展示对象
3. 区分结果物本体与提交记录
4. 判断是否受船司规则影响
5. 形成统一展示结构

## 6. 关键判断原则

- 结果物和提交记录不能混淆
- 不同结果物需要清晰归属
- 页面中要同时保留查看入口和过程记录

## 7. 和 MBL 项目的关联

服务：

- 预览件
- 草稿件
- VC件
- 提交记录
- 回执信息和截图展示

## 8. 当前版本已知不足

- 尚未形成统一命名规范
- 尚未区分文件级与动作级模型

## 9. 下一轮优化方向

- 建立结果物数据字典
- 补充不同船司下的归属差异
