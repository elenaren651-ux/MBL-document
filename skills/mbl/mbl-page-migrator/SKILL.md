# mbl-page-migrator

## 1. 目标

把现有 MBL 原型页面逐步迁移成 React 页面，同时保留业务入口、状态结构和验收基线。

## 2. 适用场景

- 从原型页迁移到 React
- 拆分页面组件
- 设计 mock 数据结构
- 规划阶段性迁移顺序

## 3. 输入

- 原型页面
- 对应 PRD
- 对应业务规则
- 现有 React 工作区状态

## 4. 输出

- 页面拆分方案
- 组件结构建议
- 状态模型建议
- mock 数据建议
- 迁移验收清单

## 5. 工作流程

1. 明确原型页的业务目标
2. 梳理页面中的状态、入口和结果物
3. 区分必须保留的业务结构与可优化的表现层
4. 设计 React 页面与组件拆分
5. 先用 mock 跑通结构，再接真实数据

## 6. 关键判断原则

- `prototype/axure-export/` 永远是对照基线
- 页面入口优先保留，链路可分阶段打通
- 先迁高频主流程，再迁异常和边角场景
- 不因后端未完成就删除业务入口

## 7. 和 MBL 项目的关联

这个 skill 直接服务：

- 列表页迁移
- 详情页迁移
- 结果物区域迁移
- 异常态与只读态迁移

## 8. 当前版本已知不足

- 尚未形成统一组件命名规范
- 尚未沉淀页面级验收模板

## 9. 下一轮优化方向

- 补充列表页和详情页的标准迁移 checklist
- 补充状态映射模板
