# mbl-field-audit-designer

## 1. 目标

围绕 MBL 制单详情页中的字段审核、冲突确认、缺失补齐和提交前确认设计交互方案。

## 2. 适用场景

- 设计字段来源候选值交互
- 设计冲突字段确认流程
- 设计缺失字段提示
- 设计审核区布局

## 3. 输入

- 字段清单
- 字段来源
- 当前审核逻辑
- 业务规则
- 错误案例

## 4. 输出

- 审核交互方案
- 字段分组建议
- 冲突字段处理策略
- 提交前校验建议

## 5. 工作流程

1. 按字段风险和频率分层
2. 识别候选值与冲突场景
3. 设计人工确认动作
4. 设计缺失值补齐入口
5. 设计提交前整体验收逻辑

## 6. 关键判断原则

- 审核界面必须支持快速判断
- 冲突字段必须展示来源
- 缺失提示要直达补齐动作
- 高风险字段优先暴露

## 7. 和 MBL 项目的关联

重点关联 `docs/prd/content/08-MBL字段来源候选值与冲突确认.md` 以及详情页审核区迁移。

## 8. 当前版本已知不足

- 尚未沉淀字段分组标准
- 尚未沉淀字段风险矩阵

## 9. 下一轮优化方向

- 建立字段审核组件规范
- 沉淀审核台典型布局
