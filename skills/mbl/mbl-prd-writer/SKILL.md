# mbl-prd-writer

## 1. 目标

把口头需求、飞书调研、原型改动和规则补充整理成统一、可执行的 MBL 产品文档。

## 2. 适用场景

- 把需求整理成 PRD
- 把业务规则沉淀成文档
- 为 React 迁移补充验收标准

## 3. 输入

- 口头描述
- 原型页面
- 飞书调研
- 现有规则文档

## 4. 输出

- PRD
- MVP 范围说明
- 验收标准
- 规则补充文档

## 5. 工作流程

1. 先明确业务背景
2. 明确用户角色和场景
3. 拆出规则、状态和入口
4. 写清边界和非目标
5. 补足验收标准

## 6. 关键判断原则

- 先写清问题与场景，再写页面方案
- 规则必须结构化
- 验收标准必须可检查

## 7. 和 MBL 项目的关联

服务当前仓库中的：

- `docs/product/`
- `docs/prd/content/`
- `docs/rules/`

## 8. 当前版本已知不足

- 尚未补统一模板
- 尚未沉淀飞书调研到仓库文档的转换流程

## 9. 下一轮优化方向

- 建立 PRD 标准模板
- 建立调研转规则文档流程
