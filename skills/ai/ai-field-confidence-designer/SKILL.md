# ai-field-confidence-designer

## 1. 目标

设计字段级置信度与候选值策略，帮助系统判断哪些字段可自动通过，哪些字段必须人工确认。

## 2. 适用场景

- 多来源字段冲突
- 字段可信度评估
- 自动通过规则设计
- 字段审核优先级设计

## 3. 输入

- 字段定义
- 不同来源的候选值
- 来源质量信息
- 已知规则
- 历史错误案例

## 4. 输出

- 字段置信度框架
- 候选值排序原则
- 自动通过条件
- 人工审核触发条件

## 5. 工作流程

1. 识别字段来源
2. 区分结构化来源与语义推断来源
3. 定义候选值优先级
4. 定义自动通过与人工确认的阈值
5. 输出前端展示和后端判断建议

## 6. 关键判断原则

- 置信度不等于正确率，但应能指导动作
- 来源质量通常比单次模型输出更重要
- 候选值展示要可解释
- 自动通过规则必须保守可控

## 7. 和 MBL 项目的关联

这个 skill 是 MBL 字段来源候选值与冲突确认能力的核心方法基础。

## 8. 当前版本已知不足

- 尚未制定统一候选值展示格式
- 尚未建立字段置信度回溯机制
