# ai-extraction-error-taxonomy

## 1. 目标

建立抽取错误分类体系，帮助团队判断问题根因，并提高优化工作的针对性。

## 2. 适用场景

- 复盘抽取错误
- 分析准确率下降原因
- 排定优化优先级

## 3. 输入

- 错误样本
- 抽取结果
- 标准答案
- 上下游日志

## 4. 输出

- 错误分类树
- 根因分析结论
- 优先修复建议

## 5. 工作流程

1. 判断是识别错误、理解错误还是规则错误
2. 判断错误是否来自输入质量
3. 判断错误是否来自候选值决策
4. 形成可统计的分类结构

## 6. 关键判断原则

- 错误分类要能指导修复，而不是只做描述
- 同一问题要能归到单一主因
- 允许保留复合原因，但要有主优先级

## 7. 和 MBL 项目的关联

服务 MBL 抽取链路中的错误复盘、评测分析和人工审核反馈闭环。

## 8. 当前版本已知不足

- 尚未建立正式分类树
- 尚未对接评测结果输出格式

## 9. 下一轮优化方向

- 补充字段级错误标签
- 补充业务严重度分层
